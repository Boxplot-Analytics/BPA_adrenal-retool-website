How to Use the Bulk Uploader

This short walkthrough will show you how to use the Bulk Uploader app in order to bulk insert many records into the database. There are two main sections:

  1. How do I create a CSV file?
  2. What happens if something goes wrong?

How do I create a CSV file?

In order to use the Bulk Uploader, you’ll need to have your data in the form of a CSV file.

NOTE: You will NOT be able to upload an Excel file through the Bulk Uploader! These are files that end with the extension .xls or .xlsx. You MUST convert your file to a CSV file before using the uploader. The new file extension will be .csv.

Fortunately, it’s extremely quick and easy to take an Excel spreadsheet and turn it into a CSV file. This section will walk you through those steps.

Check Your Spreadsheet

Before you convert your spreadsheet to a CSV file, you’ll want to check your file for to be sure that the data you’re uploading is as accurate as possible in order to avoid any errors.

At the top of this page, you’ll find a link to, test-data.csv, which is an example file that you can use to ensure your data is properly formatted.

Here are some things to keep in mind:

1. Check the ASIN. 

Each row must have an ASIN, as this is what the database will use to identify items. For new items, make sure the ASIN is unique; in other words, the ASIN cannot already be in use by another product. If you want to update an item, them check that the ASIN you have is indeed the correct one for that item.

NOTE: The Bulk Uploader will automatically overwrite the rows for any existing ASIN you upload to it. 

For example, we know that the item with the ASIN B019M554D6 already exists in the database. Here’s what that item looks like in a CSV file (truncated for explanatory purposes):

B019M554D6,"Woodstock Chimes CCSG Rainbow Makers Suncatchers Crystal Sunrise Cascade, Green",CCSG,28375270217,Active

Now, what if you uploaded a CSV file that had this row duplicated? For instance, this one:

asin,title,sku,upc,status
B000000000,"New Item to Upload",ABCD,12345678900,Active
B000000001,Another New Windchime,XYZ,12345678901,Obsolete
B019M554D6,"Woodstock Chimes CCSG Rainbow Makers Suncatchers Crystal Sunrise Cascade, Green",CCSG,28375270217,Active

The windchime that already exists in the database (the last row) is duplicated here. If you were to upload this CSV file to the Bulk Uploader, with the duplicated row, then the app would overwrite this row in the database. However, because the information in the new CSV file is exactly the same as what’s already in the database, no changes would be made in effect.

However, if you DO upload a CSV file with an existing ASIN and you’ve made changes to that item, the database WILL be changed! 

For instance:

asin,title,sku,upc,status
B000000000,"New Item to Upload",ABCD,12345678900,Active
B000000001,Another New Windchime,XYZ,12345678901,Obsolete
B019M554D6,"Woodstock Chimes CCSG Rainbow Makers Suncatchers Crystal Sunrise Cascade, Green",CCSG,28375270217,Obsolete

Notice that, now, the status of the last windchime says Obsolete. At the moment, the status in the database is Active. If you were to upload this CSV file, then because the ASIN B019M554D6 already exists in the database, the Bulk Uploader will overwrite the database to say that this item is now Obsolete.

If you want to update this item, then that’s great! Otherwise, it might be safer to delete the row you don’t want to change. Always make sure that the CSV file you’re upload only contains rows that you don’t mind changing in the database.

2. Check the status. 

This should be one of two values at the time of this writing: Active or Obsolete. Check to be sure both are spelled correctly, as Active and Atcive are viewed as two completely different things by a computer; it won’t know that this is a spelling mistake.

5. Check the data types. 

The database has very strict rules on what kinds of things you can put into the cells. For example, some columns MUST have a number in them, even if that number is 0. If there’s only an empty cell where the database is expecting an explicit number 0, the Bulk Uploader will NOT work.

Here’s a list of all columns that must have a 0 in any empty cells:

packaged__weightproduct_width
product_lengthproduct_height

Create Your CSV File

Once you’ve confirmed that your spreadsheet is in the correct format and contains only the data you want inserted or changed, it’s time to convert it to a CSV file.

In Excel, this process is extremely simple. The following excerpt from the official Microsoft documentation explains how to do this:

You can convert an Excel worksheet to a text file by using the Save As command.

  1. Go to File > Save As.
  2. Click Browse.
  3. In the Save As dialog box, under Save as type box, choose the text file format for the worksheet; for example, click Text (Tab delimited) or CSV (Comma delimited).Note: The different formats support different feature sets. For more information about the feature sets that are supported by the different text file formats, see File formats that are supported in Excel.
  4. Browse to the location where you want to save the new text file, and then click Save.
  5. A dialog box appears, reminding you that only the current worksheet will be saved to the new file. If you are certain that the current worksheet is the one that you want to save as a text file, click OK. You can save other worksheets as separate text files by repeating this procedure for each worksheet.

You may also see an alert below the ribbon that some features might be lost if you save the workbook in a CSV format.

Import or Export CSV Files | Microsoft Support

Upload Your CSV File

Now that you have your data saved as a CSV file, it’s time to upsert (update or insert) the records into the database!

  1. Navigate to the Bulk Uploader tool on the website and look for the section titled “Data to upload.”
  2. Click the grey button to upload a CSV.
  3. Select the CSV file you want to upload to the database.
  4. Check the rendered table to confirm the data was read in properly.
  5. Click the green button to upload the data to the database.

If everything goes well, the message box below this button will say “Uploading CSV file” and display a green check mark. Otherwise, it will show a red X and read, “Error processing CSV file.” If this happens, then continue on to the section below for more information on how to troubleshoot errors. Note that if your CSV file contains an error, then the query won’t execute, and no changes will be made to the database. This gives you a chance to make changes and try again, while minimizing the chances of data loss.

What happens if something goes wrong?

Sometimes, errors will pop up, even when you thought you’d taken all the necessary steps to prevent them. This section will help you to troubleshoot should any issues arise.

How will I know what went wrong?

Beneath the green button that says “Upload data,” you’ll find a message box that will tell you the status of your upload. If something goes wrong, this box will read “Error processing CSV file” and display details of the error message.

Note: If your CSV file contains an error, then the query won’t execute, and no changes will be made to the database. This gives you a chance to make changes and try again, while minimizing the chances of data loss.

These details come straight from the database, so they can be quite technical in nature. For your convenience, we’ve translated these error messages into actionable steps you can take in the event that you encounter one of them. Start by reading through the sections below. Locate the error message that’s as close to the one that you’re seeing in the message box and read the section for info on what it means and what you should do.

Error: duplicate key value violates unique constraint

What it means: This error message says that you’re trying to upload a CSV file with an item that already exists in the database. However, the Bulk Uploader should be configured to allow you to do this; the row will simply be replaced, and either overwritten with new changes or overwritten with no effective changes made. If you’re seeing this error, then that means something is wrong with the application configuration, not your CSV file.

What to do: Contact Boxplot at barb@boxplotanalytics.com and let them know you encountered this problem. There’s nothing else for you to do; this is something that can only be fixed on the back end.

Error: null value in column “asin” of relation “woodstock” violates not-null constraint

What it means: null is a programmatic word for the concept of something being missing. In other words, there’s a row in your CSV file that is missing an ASIN. It may even mean that the ASIN column is missing completely, though this is unlikely. (That’s the only column that this error should show up for—if the column name is different, please contact Boxplot at barb@boxplotanalytics.com.)

What to do: Check and make sure that your CSV file contains a column titled asin, and that every row has an ASIN in it. Don’t forget—if there’s a row with an ASIN that’s already in the database, then using the Bulk Uploader will effectively overwrite this row in the database with whatever new information you upload to it.

Error: column “bsin” of relation “woodstock” does not exist

What it means: You’re trying to upload a CSV file with a column that the database doesn’t know about. In this example, the ASIN column has a typo in it, rendering the spelling “bsin” which is not an available column. The database is highly structured, and as such your CSV file needs to follow a strict template in order to interact with it. If you’re seeing this error message, then your CSV file has an extra column that’s not in the database.

What to do: Remove the column from your spreadsheet and try to upload the data again.

Error: invalid input syntax for type integer: “”

What it means: One of your columns requires a numeric value to be in the cell, but you either have a word there instead or nothing.

What to do: Make sure that all columns that are numeric in nature have a 0 in each empty cell. Otherwise, the Uploader will not be able to parse your file.

packaged__weightproduct_width
product_lengthproduct_height
inner_pack_qtypackaged_width
moqpackaged__length
case_pack_qtyitems_per_inner_pack
